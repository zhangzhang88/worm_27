# Zeabur 部署详细步骤指南

以下是在 Zeabur 上部署后端服务的完整步骤：

---

## **步骤 1：登录 Zeabur**
1. 打开浏览器，访问 [Zeabur 官网](https://zeabur.com/)
2. 使用 GitHub 账号或其他方式登录 Zeabur

---

## **步骤 2：创建或选择项目**
1. 如果是首次使用，点击「创建新项目」按钮
2. 为项目命名（如 `worm_backend`）并点击「创建」
3. 如果已有项目，直接选择对应的项目

---

## **步骤 3：添加后端服务**
1. 在项目界面中，点击「添加服务」按钮
2. 在弹出的选项中选择「从 GitHub 导入」

---

## **步骤 4：连接 GitHub 仓库**
1. 如果是首次连接，需要授权 Zeabur 访问您的 GitHub 账号
2. 授权完成后，选择包含 `worm_27` 代码的仓库
3. 选择要部署的分支（通常是 `main` 分支）
4. 点击「下一步」

---

## **步骤 5：配置服务参数**
1. **根目录设置**：
   - 在「根目录」输入框中填写 `backend/`
   - 这一步非常关键，确保 Zeabur 能找到我们的后端代码

2. **环境变量（可选）**：
   - 如果您的应用需要环境变量，可以在此处添加
   - 目前我们的后端应用不需要额外环境变量，可跳过

3. **端口设置**：
   - Zeabur 会自动检测应用端口
   - 我们的后端应用默认使用 `5001` 端口，Zeabur 会自动适配

4. 点击「部署」按钮开始部署流程

---

## **步骤 6：查看部署状态**
1. 部署过程中，您可以在服务界面看到实时日志
2. 等待部署完成，状态会显示为「运行中」

---

## **步骤 7：验证部署成功**
1. 点击服务卡片上的「访问 URL」查看应用状态
2. 尝试访问 `[访问URL]/health`，如果返回 `{"status": "healthy"}` 则说明部署成功

---

## **常见问题排查**
### 1. 部署失败提示「根目录不存在」
   - 检查根目录是否填写为 `backend/`（注意结尾的斜杠）
   - 确保 `backend/` 目录在 GitHub 仓库中已正确提交

### 2. 依赖安装失败
   - 检查 `requirements.txt` 文件是否包含所有必要依赖
   - 确保依赖版本兼容 Python 3.9+

### 3. 应用启动失败
   - 查看部署日志，检查是否有端口冲突或其他错误信息
   - 确保 `app.py` 中的 `host='0.0.0.0'` 设置正确

---

## **文件说明**
- `backend/requirements.txt`：后端应用依赖列表
- `backend/Procfile`：Zeabur 启动命令配置
- `backend/app.py`：后端应用主入口
